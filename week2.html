Dirty Data

In class we learned how data often needs to reorganized and standardized after getting it from the source. The example we had to shift through was of that of the incident of the titanic. There were multiple columns with great information, but also many columns that needed to be split and broken up in order for computers to understand the information or for us to make proper charts. There was missing information, and of course there was the question of accuracy.


Even though the data was imperfect, there was also value that we could gleam from the data. Average of male/female survivors, average age of survivors and the ratio of class level to surviver numbers. We also learned about Broman’s principles of data organization. How clarity, consistency and simplicity really help to make data better. Using YYYY-MM-DD format for dates helps to use the data from excel to most data processing programs. Not using excel formatting, highlights or any extra calculations is also helpful for computers to translate. Saving data in plain text files ensures the that data will not need any special software to open but can still be opened in excel.


Data dictionaries are incredibly useful, to help give a reference point for analyst or any other person easily see the data fields at quick glance. Putting a description and units of measurement really helps to make the intent clear, while leaving the data itself simple. Putting data in rectangles helps organize the data, instead of separating by week, etc put the same type of data side by side with a delimiter describing the difference. Broman’s example was that of a heading of week 1 reorganized and situated into a heading of date_1 weight_4 glucose_4. So the data and week were connected by the field box, not two separate field boxes.

Mostly in class we learned to keep the data fun and interesting but to also be diligent in cleaning up our data, making sure it is from accurate reputable sources so we can create well crafted data visualizations.
